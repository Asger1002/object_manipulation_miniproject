# Finger Tip LUT NPZ Guide

This file documents how the finger tip lookup table is stored in `finger_tip_lut.npz`, what each entry means, and how to use it.

## Purpose

The LUT stores precomputed 4x4 homogeneous transforms (SE3) for finger tips so applications can:

- Avoid repeated forward kinematics for common queries
- Perform fast lookup for visualization and planning
- Compose thumb motion from independent flexion and opposition transform sets

The LUT is generated by `generate_lut()` in `model.py`.

## File Location

By default, the generator writes:

- `laptop_ws/grasping/finger_tip_lut.npz`

The verifier reads the same file in `verify_lut.py`.

## NPZ Contents

The NPZ is a dictionary-like container with these keys:

- `resolution`: scalar integer sample count per motion axis
- `middle`: shape `(resolution, 4, 4)`
- `ring`: shape `(resolution, 4, 4)`
- `little`: shape `(resolution, 4, 4)`
- `index_flex`: shape `(resolution, 4, 4)`
- `thumb_flex`: shape `(resolution, 4, 4)` relative to neutral thumb pose
- `thumb_opposition`: shape `(resolution, 4, 4)` relative to neutral thumb pose

All 4x4 matrices are homogeneous transforms with final row approximately `[0, 0, 0, 1]`.

## Sampling Convention

Samples are uniformly spaced over normalized control range [0, 1].

For resolution = N:

- sample index i maps to control value i / (N - 1)
- sample_0 is neutral for all channels
- sample_(N-1) is max for that channel

## Motion Assumptions Encoded

- MRL fingers (middle, ring, little) are coupled and sampled together from MRL flexion
- Index flexion is sampled from the TISIT positive branch used for index closing
- Thumb flexion and thumb opposition are stored separately as relative transforms

Important: thumb opposition is mechanically/control-coupled in the overall hand model, but storing separate relative thumb transforms still allows correct composition.

## Thumb Composition

If `T_flex_rel` and `T_opp_rel` are read from the NPZ, the tested correct composition order for matching direct FK is:

- `T_thumb_rel = T_opp_rel @ T_flex_rel`

This order was validated in `verify_lut.py` by comparing composed transforms against direct forward kinematics from `model.py`.

## Typical Usage

1. Load NPZ once at startup.
2. Convert normalized control values to sample indices.
3. Read per-finger transform for fast query.
4. For thumb, compose opposition and flexion relative transforms in the validated order.
5. If needed, multiply by neutral/world baseline transform in your application frame setup.

## Minimal Loading Example

```python
import numpy as np

lut = np.load("finger_tip_lut.npz")
N = int(lut["resolution"])

# Example lookup
i = 5
t_middle = lut["middle"][i]

# Example thumb composition
i_flex = 3
i_opp = 5
t_thumb_rel = lut["thumb_opposition"][i_opp] @ lut["thumb_flex"][i_flex]
```

## Recommended Validation Steps

After regenerating the LUT:

1. Run `verify_lut.py`
2. Confirm SE3 checks pass
3. Confirm thumb composition-vs-FK check still reports "opposition then flex"

## Common Uses

- Real-time fingertip pose lookup in grasp planners
- Fast rollout of candidate grasps without full FK each step
- Dataset generation and augmentation for learning pipelines
- Visual analytics of reachable fingertip trajectories

## Notes

- If joint limits, URDF geometry, or control mapping changes, regenerate the LUT.
- Keep generator (`model.py`) and verifier (`verify_lut.py`) in sync with format assumptions.
